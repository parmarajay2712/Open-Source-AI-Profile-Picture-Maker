## 🔄 How It Works

### System Architecture

```
                    ┌──────────────────────────────┐
                    │         Nginx (Prod)          │
                    │    Static Frontend Serving    │
                    └─────────────┬────────────────┘
                                  │
           ┌──────────────────────▼─────────────────────┐
           │              React Frontend                  │
           │   Upload → Select Action → Poll → Download   │
           └──────────────────────┬─────────────────────┘
                                  │ HTTP REST
           ┌──────────────────────▼─────────────────────┐
           │           Express.js Backend API             │
           │    Multer Upload  ·  BullMQ Queue  ·  GC    │
           └──────┬───────────────────────────┬──────────┘
                  │                           │
     ┌────────────▼──────┐        ┌───────────▼───────────┐
     │   Redis (BullMQ)  │        │  Temp Filesystem       │
     │   Job Queue Store │        │  (UUID-named files)    │
     └───────────────────┘        └───────────────────────┘
                  │
     ┌────────────▼──────────────────────────────────────┐
     │              AI Microservices (Docker)             │
     │                                                    │
     │  ┌─────────────┐  ┌────────────┐  ┌────────────┐  │
     │  │ rembg :8001 │  │ GFPGAN     │  │ Stable Diff│  │
     │  │ Background  │  │ :8002 Face │  │ :8003 Style│  │
     │  │ Removal     │  │ Enhancement│  │ Transfer   │  │
     │  └─────────────┘  └────────────┘  └────────────┘  │
     └───────────────────────────────────────────────────┘
```

### 🖼️ Image Processing Flow

```
  ┌──────────┐    ┌───────────────┐    ┌──────────────────┐
  │  Upload  │───▶│ Backend saves │───▶│ jobId returned   │
  │  Selfie  │    │ to temp/      │    │ to frontend      │
  └──────────┘    └───────────────┘    └──────────────────┘
                                                │
                                                ▼
  ┌────────────┐   ┌──────────────┐   ┌─────────────────┐
  │  Download  │◀──│   Frontend   │◀──│  BullMQ Worker  │
  │  PNG file  │   │ polls status │   │ calls AI service│
  └────────────┘   └──────────────┘   └─────────────────┘
                                                │
                                                ▼
                                   ┌─────────────────────┐
                                   │  Auto-cleanup after  │
                                   │  1 hour via GC job   │
                                   └─────────────────────┘
```

### 🔗 Job Chaining

```
  Original Upload
        │
        ▼
  ┌─────────────┐      ┌─────────────────┐      ┌──────────────┐
  │  Remove BG  │─────▶│  Enhance Face   │─────▶│  Apply Style │
  │  (rembg)    │      │  (GFPGAN)       │      │  (Stable Diff│
  └─────────────┘      └─────────────────┘      └──────────────┘
        │                      │                        │
        └──────────────────────┴────────────────────────┘
                               │
                         Each operation uses the
                         LATEST result as its input.
                         Chain as many as you want.
```

### 🔄 Async Processing (BullMQ)

```
  Request comes in
        │
        ▼
  Job added to Redis queue
        │
        ├──▶ Redis available?  ─── YES ──▶ Worker picks up job async
        │
        └──▶ Redis unavailable? ──▶ Synchronous fallback (slower)
```

```bash

# 3. Open in browser
```

## 📖 Usage Guide

<details open>
<summary><strong>✂️ Remove Background</strong></summary>

1. Drag & drop or click to upload a selfie (JPG/PNG, max 10MB)
2. Click **"Remove Background"**
3. Wait 5–15 seconds for `rembg` to process
4. Result shows with transparent background
5. Proceed to replace background or download

</details>

<details>
<summary><strong>🎨 Replace Background</strong></summary>

1. After background removal, open the **Background Tool**
2. Choose a mode:
   - **Color** — pick from presets or enter a hex code
   - **Image** — upload a custom background photo
   - **Blur** — adjust blur radius (1–20)
3. Click **"Replace Background"**

</details>

<details>
<summary><strong>✨ Enhance Face</strong></summary>

1. Click **"Enhance Face"** (works on any image)
2. GFPGAN v1.4 restores and upscales facial details
3. Works best on clear, front-facing photos
4. Result replaces the current canvas image

</details>

<details>
<summary><strong>🖌️ Apply AI Style</strong></summary>

1. Click any style in the **Style Grid** (Professional, Anime, Cyberpunk, etc.)
2. Optionally enter a **custom prompt** to guide the AI
3. Click **"Apply Style"**
4. Stable Diffusion + ControlNet generates your stylized portrait

</details>

<details>
<summary><strong>🔗 Chaining Operations</strong></summary>

You can chain multiple operations! Recommended flow:

```
Upload → Remove BG → Enhance Face → Apply Style → Replace BG → Download
```

Each step uses the output of the previous as its input.

</details>

### 💡 Tips for Best Results

- Use images of **at least 512×512px**
- Ensure **good, even lighting** in the original photo
- For face enhancement, use a **clear front-facing portrait**
- For style transfer, **simpler backgrounds** produce better results

## 🗄️ Data Storage

### Design Philosophy: No Database

```
┌─────────────────────────────────────────────────────────────┐
│  PFP Maker is intentionally stateless. There is no user     │
│  database. No accounts. No persistent records.               │
│                                                              │
│  All data lives in:                                          │
│    • temp/ folder → image files (UUID-named)                 │
│    • In-memory job map → status & metadata                  │
│    • Redis → job queue only (not data persistence)           │
│                                                              │
│  Everything is cleaned up automatically after TTL expires.   │
└─────────────────────────────────────────────────────────────┘
```

### In-Memory Job Object

```typescript
interface Job {
  id:           string;               // UUID
  status:       'pending' | 'processing' | 'completed' | 'failed';
  originalPath: string;               // Path to uploaded image
  resultPath:   string | null;        // Path to processed result
  progress:     number;               // 0–100
  error:        string | null;        // Error message if failed
  createdAt:    number;               // Unix timestamp
  updatedAt:    number;               // Unix timestamp
}
```

### BullMQ Queue Job Payload

```typescript
interface QueueJob {
  jobId:    string;          // Reference to in-memory job
  action:   string;          // 'remove-bg' | 'enhance' | 'apply-style' | 'replace-bg'
  preset?:  string;          // Style preset ID
  prompt?:  string;          // Custom prompt
  bgType?:  string;          // Background type
  bgValue?: string;          // Background value
}
```

### File Lifecycle

```
Upload → UUID filename assigned → Processing → Result saved → TTL expires → Deleted
  0s          instant              5–30s         instant         3600s       auto
```

## 🔒 Security

### Measures Implemented

| Layer | Mechanism | Detail |
|-------|-----------|--------|
| 🛡️ **HTTP Headers** | Helmet.js | CSP, HSTS, X-Frame-Options, XCTO |
| 🚦 **Rate Limiting** | express-rate-limit | Per-IP throttling on all endpoints |
| 🌐 **CORS** | Whitelist-only | Only `FRONTEND_URL` allowed |
| 📂 **File Validation** | Multer + type check | JPEG/PNG only, 10MB max |
| 🧹 **Auto-Cleanup** | Cron GC service | Files deleted after TTL |
| ❌ **No Auth Surface** | By design | No login = no credential attacks |
| 📊 **No PII Storage** | By design | No user data ever stored |

### Security Checklist

- [x] 🛡️ Helmet.js — full HTTP security header suite
- [x] 🚦 Rate limiting + IP throttling on all endpoints
- [x] 🌐 CORS hardened — whitelist-only origins
- [x] 📂 File type & size validation on upload
- [x] 🧹 Automatic file cleanup — no orphaned data
- [x] ❌ No user database — zero credential attack surface
- [x] 📊 No PII collected or stored anywhere
- [x] 🔇 Generic error messages — no stack traces exposed

## 📊 Performance & Scalability

<div align="center">

| Metric | Notes |
|:------:|-------|
| ⚡ **rembg processing** | ~5–10s (CPU) / ~1–2s (GPU) |
| 🪄 **GFPGAN enhancement** | ~10–20s (CPU) / ~2–5s (GPU) |

| 🔄 **Concurrent jobs** | 2 workers (configurable via BullMQ) |
| 📈 **Horizontal scaling** | AI services scale independently |
| 🌐 **Frontend** | Fully static — serve from any CDN |
| 🗄️ **No DB bottleneck** | Stateless design scales linearly |

</div>

### Scaling Strategy

```
For high traffic:

Frontend      →  Deploy to CDN (Vercel, Cloudflare Pages)
Backend       →  Horizontal scale behind load balancer
Redis         →  Redis Cluster for queue resilience
AI Services   →  Scale each service independently by demand
               (rembg gets more traffic than Stable Diffusion)
```
